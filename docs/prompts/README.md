# Prompts Operativos

Guarda aqui los prompts versionados por rol o por frente de trabajo.

## Convencion
- `hierarchy.md`: jerarquia operativa y niveles de responsabilidad.
- `orchestrator.md`: prompt del agente orquestador principal.
- `backend-events.md`: prompt del especialista backend y eventos.
- `frontend-ops.md`: prompt del especialista del portal corporativo.
- `infra-sre.md`: prompt del especialista de infraestructura y CI/CD.
- `security.md`: prompt del especialista de seguridad.
- `ui-brand.md`: prompt del especialista visual.
- `mobile-edge.md`: prompt del especialista mobile offline-first.
- `reviewer.md`: prompt del revisor tecnico principal.
- `doc-steward.md`: prompt del guardián de documentacion.

## Regla practica
- Un prompt por archivo.
- Leer `hierarchy.md` antes de invocar un rol.
- Mantenerlos cortos, precisos y orientados a entregables.
- Si cambia la arquitectura o el flujo, actualizar primero el prompt y luego la documentacion asociada.
